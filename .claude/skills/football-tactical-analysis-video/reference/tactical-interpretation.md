# 战术解读:先核实,再拆解

战术解析片的价值=把"球为什么进"讲明白。最大的坑是**只看画面猜过程**——720p 集锦里球员只有~20px,谁射门、怎么形成的,光看画面极易判错(本管线首作就把一场帽子戏法的三个进球射手/机制**全判错**过:误以为是另一名球员首开纪录、且套了三种错误打法)。所以流程永远是**先核实事实,再写解读**。

## 铁律:用英文原解说核实事实
1. **只下英文原解说字幕**(`--skip-download`=不重复下整场视频,只要字幕;比赛画面另外下/已有):
   ```bash
   yt-dlp --write-auto-subs --sub-langs "en.*" --skip-download \
     --extractor-args "youtube:player_client=tv_embedded,web_embedded,android_vr,mweb" \
     "<比赛/集锦 URL>"
   ```
   (嵌入式客户端参数绕 PO-token;**下整场画面**用同样参数但去掉 `--skip-download`、整段下不要 `--download-sections`。)
2. **按进球时间戳读解说词**——解说会**喊出射手名、助攻、过程**(谁传谁射、第几落点、撞墙还是直塞)。以解说为准,画面只用来对齐镜头时间点。
3. 可借鉴 **CBS / Tifo Football / 懂球帝等战术解读**(同法下其字幕)补术语与背景,但事实仍以原场解说为准。

## 结构:主题脊梁 + 每球两段
- **主题脊梁**:全片一个论点(如"同一个终结者,三种方式破密集防守")。`intro` 抛出,`ending` 回扣。每粒进球都是这个论点的一个论据。
- **每球两段**(对应 project.json 的两个 segment):
  - **拆解·导火索**(实时 build-up 画面):进攻怎么发起、**关键决策点**是什么。口播在真实推进画面上讲清楚。
  - **进球**(直接切进球慢镜头 speed≈0.55):喊出终结,并给这粒球**命名一个机制标签**(下表)。

## 机制词汇表(给每球贴一个主导标签)
破密集防守 / 运动战常见机制,挑**最主导**的一个当这粒球的"钥匙",解说和 overlay 都围绕它:

| 标签 | 含义 |
|---|---|
| 纵向渗透 | 纵向直塞打穿中路,不横传倒脚 |
| 撞墙配合 | 二过一 / 做球回做 |
| 二次进攻 | 射门被扑/挡出,抢第二落点补射 |
| 无球跑动 | 不靠盘带,靠跑位出现在正确时间正确地点 |
| 边中结合 | 边路传中找中路包抄 |
| 肋部渗透 | 肋部"口袋区域"做球后插上 |
| 宽度拉扯 | 一侧拉开宽度,给另一侧/中路腾空间 |
| 快速反击 / 定位球 | 转换速度 / 战术角球任意球 |

一粒球常多机制叠加(如"边中结合 + 无球内插"),解说可都提,但**标签只选最致命的那个**,主题脊梁才清晰。

## 画面与事实对齐
- 进球慢镜头 `footage.start` 用解说时间戳定位的进球瞬间(可先抽缩略图墙 `tile=4x3` 精确定位)。
- build-up 段口播讲的决策点,要和实时画面**对得上**(别讲了画面里没有的东西)。
- 判别球队/球员时,**避开对手特写镜头**造成的误读。
- 写完解说,再回看一遍:每句话画面里都能找到对应吗?对不上就改解说或改 footage 时间点。
