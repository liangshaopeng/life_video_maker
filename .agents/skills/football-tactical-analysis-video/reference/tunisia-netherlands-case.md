# 案例: 突尼斯1-3荷兰战术解析

## 适用场景
当用户要做“完整比赛/长集锦 → 竖屏足球战术解析”,且希望有阵容介绍、进球拆解、抖音/视频号/B站风格 BGM 时,先按本案例的取舍做第一版。

## 本案定版结构
1. **阵容展示·突尼斯**:直接切转播阵容图,约 10~12 秒。只讲 `5-3-2` 的战术特点:三中卫守禁区、翼卫先退、中场封肋部。
2. **阵容展示·荷兰**:继续切转播阵容图,约 10~12 秒。只讲 `4-3-3` 的战术特点:边锋/边后卫拉宽、布罗比占中路、范戴克抢定位球第一点。
3. **第一球拆解**:阵容后立刻进入第一球机制,不要再加普通概括段。
4. **每球两段**:build-up 解释决策点,goal 用慢镜头看清射门/折射/球进网。
5. **收尾**:回扣主线:低位防线怕早传逼转身,定位球怕第一步抢点。

## 开场镜头规则
- 不要用一般比赛镜头开场。用户会觉得“吸引力一般”。
- 优先级:
  1. 转播阵容图/官方阵型图。
  2. 本场 MVP 或核心球员的高光特写/进球动作。
  3. 其他镜头都不适合做第一秒。
- 如果用户说“前面太短”,不是恢复冗长概括,而是补足阵容展示感:两队各 10 秒左右,画面必须是阵容图或球员展示。

## 文案节奏
- 阵容段只讲特点,不念完整首发。
- 示例:
  - `开场直接看阵容。突尼斯五三二守禁区,核心就是三中卫站住禁区,两名翼卫先退回来,中场三人堵住肋部。`
  - `再看荷兰。四三三不是硬往中路撞,边锋和边后卫把宽度拉满,布罗比顶住中卫,范戴克留着定位球抢第一点。`
- 语速可用 `zh-CN-YunjianNeural rate +30%`,激情靠语速和重音,不要靠一堆感叹号。

## BGM 选择
- 用户不满意“泛史诗/电影感”时,要搜中文互联网里的足球剪辑语境,不要只靠本地曲库直觉。
- 本案最终选 `Attraction - Masazumi Ozawa`,中文页面常标为“天下足球/每周10佳进球背景音乐”。它比 `Clash_Defiant` 这类通用史诗曲更像足球集锦。
- 处理步骤:
  1. 搜索关键词: `抖音 足球剪辑 BGM 推荐`, `足球集锦 BGM Attraction`, `CCTV5 足球进球集锦 BGM`。
  2. 优先选“足球进球集锦/天下足球/十佳球”语境明确的歌。
  3. 下载后用 `ffprobe` 查时长,防止只拿到 5 秒试听片。
  4. 用 `mix_bgm.sh vertical.mp4 bgm.mp3 final.mp4 0 <音量>` 混音;如果最后 loudnorm 后数值变化不大,仍可通过提高 BGM 输入音量增强存在感。
- 版权提醒:这些热门 BGM 多半会触发平台版权审核,交付时要提醒用户。

## 中文球员名
- 不确定外援/欧洲球员中文名时,查中文互联网常用叫法再定稿。
- 本案:
  - Jan Paul van Hecke 用“范赫克”,不要用“范黑克”。
  - Virgil van Dijk 用“范戴克”,但不能把范赫克的进球说成范戴克。
  - Hannibal Mejbri 首次可写“汉尼拔梅布里”,后续可写“汉尼拔”。

## 验证清单
- `python3 make_narration.py project.json`:确认阵容段各约 10~12 秒。
- `python3 render_overlays_v.py project.json && python3 assemble_v.py project.json`。
- `bash mix_bgm.sh ...` 重新混最终版。
- `python3 verify_goals.py project.json`:确认进球段仍看得见“射门/触球→球进网”,段数变化后要更新 `build/gverify`。
- `ffmpeg -ss 0.5 -i final.mp4 -frames:v 1 build/opening_check.jpg`:第一帧附近必须是阵容图或核心高光。
- `ffprobe` 验证 `1080x1920 / 30fps / 时长合理`。
